Timmers ATS — Applicant Tracking System Migration

Role: Freelance Full-Stack Developer
Client: Sophia
site:https://www.fleks.works/


The Problem

The client needed to migrate off their existing Fleks-based applicant tracking system onto a new, custom-built ATS ("Timmers") — without losing any functionality reps and hiring owners already depended on.

What I Built

Phase 1 — Full Parity Migration


Custom Form Editors — rebuilt the form editing functionality reps use to configure application flows, matching the original system's flexibility
29 Synced Job Functions — migrated and synced all 29 job functions from the legacy Fleks system, preserving existing data and workflows
Owner-Based Distribution System — built a new distribution system that routes applications and job data based on ownership, configured for the client's hiring owners
Website Embeds — prepared the system for embedding directly into the client's website, ready for the next phase of rollout


Outcome

Delivered 100% feature parity with the legacy Fleks system — a stable, fully functional replacement, embedded on the client's website with zero disruption to existing hiring workflows. Project completed and delivered.
